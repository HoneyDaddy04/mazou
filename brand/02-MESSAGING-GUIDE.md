## 1. Voice Principles

### Direct
Say it plainly. No buzzwords, no filler. The product is good enough to speak for itself.

### Confident, Not Arrogant
We know the problem. We built the fix. But we don't need to shout about it.

### Technical but Accessible
Developers are our primary audience. But a CTO, PM, or finance lead should understand every line of copy too.

### African, Not "African-Themed"
We don't perform Africanness. We are African by solving African problems with African context. No flags, no sunset imagery, no "empowering the continent" platitudes.

## 5. Messaging by Audience

### For Developers

**Key message**: It's the API you already know, but cheaper and in your currency.

**Language to use**:
- "Drop-in replacement for OpenAI SDK"
- "One line change in your code"
- "model='auto' and we handle routing"
- "Tag your calls, see per-feature costs"
- "BYOK  - bring your existing API keys"

**Language to avoid**:
- "No-code solution"
- "Easy to use" (patronizing)
- "Democratizing AI" (vague)

**Proof points**:
- OpenAI-compatible API
- 5-minute integration
- Python + JS SDKs
- Smart routing saves 15-40%

---

### For CTOs / Engineering Leaders

**Key message**: Full visibility and control over your AI spend, without changing your stack.

**Language to use**:
- "Per-feature, per-model cost attribution"
- "One invoice instead of ten"
- "No vendor lock-in  - BYOK supported"
- "Role-based access for your team"
- "Audit trail for every API call"

**Language to avoid**:
- "Cutting-edge" (meaningless)
- "AI-powered" (ironic)
- "Best-in-class"

**Proof points**:
- Dashboard with spend-by-feature breakdown
- Optimization recommendations with projected savings
- 40+ models, one integration
- Local currency = no FX surprises

---

### For Finance / CFOs

**Key message**: Your AI costs are in naira, predictable, and visible.

**Language to use**:
- "One monthly naira invoice"
- "Know what every AI feature costs before the bill arrives"
- "Prepaid wallet  - never overshoot budget"
- "40% average cost reduction through smart routing"
- "No USD cards, no FX, no surprises"

**Language to avoid**:
- Technical jargon (API, SDK, endpoint)
- Model names (GPT, Claude, etc.)
- Developer-facing copy

**Proof points**:
- Paystack integration (trusted rails)
- Per-feature cost breakdown
- Budget alerts and controls
- Prepaid model = spend only what you fund

---

### For Investors

**Key message**: Mazou is the infrastructure layer for Africa's AI economy  - Paystack for AI.

**Language to use**:
- "500K+ developers in Nigeria, growing 50%+ annually"

## 7. Key Phrases & Terminology

### Always Say | Never Say

| Always | Never |
|---|---|
| "AI gateway" | "AI platform" (too vague) |
| "Smart routing" | "AI-powered routing" (redundant) |
| "Local currency" | "Localized billing" (corporate) |
| "Cost visibility" | "Analytics dashboard" (generic) |
| "African models" | "Local models" (ambiguous) |
| "Fund your wallet" | "Top up your account" (informal) |
| "Per-feature costs" | "Granular analytics" (jargon) |
| "Drop-in replacement" | "Seamless migration" (corporate) |
| "One API key" | "Unified access point" (corporate) |

### Product Terminology

| Term | Definition | Notes |
|---|---|---|
| **Smart routing** | Automatic model selection based on cost, quality, and language | Always lowercase "smart" |
| **Managed mode** | Mazou holds provider API keys; user pays in naira | Primary offering |
| **BYOK** | Bring Your Own Keys  - user provides their API keys | Always caps, always explain on first use |
| **Signal dot** | The period in the mazou. wordmark | Internal brand term |
| **Wallet** | Prepaid naira balance for API usage | Not "credits", not "tokens" |

## 8. Content Rules

1. **Lead with the benefit, not the feature.** "Save 40%" not "Our routing engine uses multi-factor optimization."
2. **Use numbers.** "40+ models" not "many models." "15-40% savings" not "significant savings."
3. **Short sentences.** If a sentence has a comma, consider splitting it.
4. **Active voice.** "Mazou routes your request" not "Your request is routed by Mazou."
5. **No exclamation marks** in product copy. Ever.
6. **One CTA per section.** Don't ask the reader to do two things at once.
7. **Code is copy.** A three-line code sample often communicates better than a paragraph.
